01 / Der vollständige Workflow

Von Figma
bis GitHub

Das ist der komplette Kreislauf – Schritt für Schritt, Tool für Tool:

Du
Du · Figma
Du designst eine UI Komponente
Ein Button, eine Navigation, eine Karte – was auch immer. Du arbeitest in Figma wie immer. Keine Änderung in deinem Design-Prozess.
MCP
Model Context Protocol
MCP verbindet Figma mit Claude
MCP ist die Brücke – ein Protokoll das Claude erlaubt direkt in Figma zu schauen. Claude sieht dein Design: Farben, Abstände, Schriften, Struktur.
figma_get_design_context → Claude liest dein Design
CC
Claude Code
Claude schreibt den Code
Claude Code liest das Figma Design und generiert HTML, CSS – oder React, Vue, was immer du brauchst. Pixel-genau, mit deinen echten Design-Tokens.
"Convert this Figma frame to HTML and CSS"
CC
Claude Code · Git
Claude committet und pusht
Claude Code führt automatisch git add, git commit und git push aus – mit einer sinnvollen commit message. Du tippst keinen einzigen Git-Befehl.
"Commit these changes and push to GitHub"
GH
GitHub
Code landet auf GitHub
Dein Code ist versioniert, gesichert, und zugänglich. Jede Änderung hat eine commit message, eine ID, einen Zeitstempel. Die vollständige Historie deines Projekts.
Du
Du · Terminal
Du pullst und prüfst
Mit einem einzigen Befehl holst du alle Änderungen auf deinen Mac. Du schaust, bewertest, und gibst Claude neue Anweisungen wenn nötig.
git pull → alles lokal aktuell
02 / Die vier Tools

Was jedes Tool
tut

Figma
Dein Design-Tool
Wo du designst. Unverändert. Figma bleibt Figma – nur dass Claude jetzt direkt reinschauen kann.
Claude Code
Dein KI-Assistent
Versteht dein Design, schreibt den Code, führt Git-Befehle aus. Du sprichst mit ihm auf Englisch wie mit einem Kollegen.
MCP
Die Brücke
Model Context Protocol – der Standard der Claude mit externen Tools wie Figma, GitHub, und anderen verbindet. Unsichtbar, aber entscheidend.
GitHub
Dein Gedächtnis
Speichert alles versioniert in der Cloud. Jede Änderung, jeder Commit, die gesamte Geschichte deines Projekts – sicher und zugänglich.
03 / Die Rollen

Du und Claude –
wer macht was

Deine Rolle
Der Dirigent
  • UI in Figma designen
  • Claude Anweisungen geben
  • Ergebnisse prüfen und bewerten
  • Richtung und Vision bestimmen
  • Finale Entscheidungen treffen
Claude's Rolle
Das Orchester
  • Figma Design lesen und verstehen
  • HTML / CSS Code schreiben
  • Git add, commit, push ausführen
  • Fehler erkennen und korrigieren
  • Mehrere Dateien gleichzeitig bearbeiten
Das Wichtigste

Du musst kein Entwickler werden. Aber du musst verstehen was passiert – Git, GitHub, Branches, Commits. Das hast du bereits gelernt. Claude übernimmt die Ausführung, du behältst die Kontrolle.
